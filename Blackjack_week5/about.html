About & Project Documentation

Scope, design, and development plan for Royal Blackjack.

Project Overview

Royal Blackjack is an interactive browser-based card game built as a progressive web development project across five weeks. The application recreates the classic casino game of Blackjack (21), allowing a single player to compete against an automated dealer.

The project is developed incrementally, starting with semantic HTML5 structure (Weeks 1–2), styled with CSS3 (Weeks 2–3), and brought to life with JavaScript game logic (Weeks 4–5).

Project Scope

In Scope

  • Four interconnected HTML pages (Home, Game, Rules, About)
  • Full Blackjack game logic (Hit, Stand, Double Down)
  • Animated card dealing and score calculation
  • Chip-based betting system with running balance
  • Round history log table
  • Responsive design for desktop, tablet, and mobile
  • Accessibility: ARIA roles, keyboard navigation, semantic HTML

Out of Scope

  • Real-money transactions or cryptocurrency
  • Multiplayer / server-side networking
  • User account creation or persistent database storage
  • Card splitting (may be added as a stretch goal)
  • Mobile app packaging (PWA manifest optional stretch goal)

Stretch Goals

  • Split pairs action
  • LocalStorage-based high score persistence
  • Card flip CSS animations ✅ Completed Week 3
  • Sound effects (deal, win, bust)

Design Document

Visual Direction

The aesthetic is Modern Dark + Neon Glow — deep void black background, neon cyan/magenta/gold accents, and crisp white card faces with glow effects. The design language draws from underground casino culture meets cyberpunk, using CSS3 animations to bring the interface to life.
